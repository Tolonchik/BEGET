Бегущая строка Бегущая строка – это анимационный эффект, при котором строка текста перемещается по странице один раз, несколько раз или постоянно. Для создания бегущей строки используется специальный тэг

MARQUEE>
. Атрибуты этого тэга контролируют параметры перемещения текста. 1) BEHAVIOR – способ выполнения эффекта бегущей строки: a. Scroll –текст исчезает за краем страницы; b. Slide – после выполнения заданного числа циклов, текст остается у левого или правого поля страницы; c. Alternate – направление перемещения текста в строке меняется на противоположное, после выполнения каждого цикла. По завершению последнего цикла текст остается у края страницы. 2) BGCOLOR – цвет фона бегущей строки. 3) DIRECTION – направление перемещения текста: a. Left – влево; b. Right – вправо. 4) HEIGHT – высота бегущей строки. 5) HSPACE – отступ в пикселях текста бегущей строки от левого и правого полей. 6) LOOP – число показов текста в строке: a. ? – целочисленное значение, указывающее число повторов; Infinite – бесконечное повторение эффекта по циклу. 7) SCROLLAMOUNT – смещение текста в пикселях за один шаг. 8) SCROLLDELAY – временной промежуток между смещениями в миллисекундах, по умолчанию 60 мс. 9) VSPACE – отступ в пикселях по вертикали от текста до рамки бегущей строки. 10) TRUESPEED – минимальное значение смещения текста, по умолчанию 60 мс. 11) WIDTH – ширина бегущей строки в пикселях. 7. Разметка web – страниц при помощи фреймов Технология фреймов позволяет разделить Web – страницу на секции для отображения в окне обозревателя сразу нескольких документов HTML. Фреймовая верстка предоставляет разработчикам Web – страниц ряд преимуществ: 1) Документы, отображаемые в фреймах, независимы друг от друга. Прокручивание содержимого в одном фрейме не влияет на содержимое в другого. 2) Гиперссылки в одном фрейме могут изменять содержимое другого. Все размеры указываются через запятую в пикселях или в процентах от размера окна обозревателя. CSS Что такое таблицы стилей Каскадные (многоуровневые) таблицы стилей - cascading style sheets (CSS) — это мощный стандарт на основе текстового формата, определяющий представление данных в обозревателе. Если формат HTML предоставляет информацию о составе документа, то таблицы стилей сообщают, как он должен выглядеть. Таким образом, каскадные таблицы стилей дают возможность хранить содержимое отдельно от его представления. Стиль включает все типы элементов дизайна: шрифт, фон, текст, цвета ссылок, поля и расположение объектов на странице. Методы и синтаксис Существует ряд методов, с помощью которых таблицы стилей могут применяться к документу HTML. Синтаксис соответствует реальной структуре информации, содержащейся внутри таблицы стиля. Методы Существует три метода для применения таблицы стилей к документу HTML: o Встроенный (Inline). Этот метод позволяет взять любой тег HTML и добавить к нему стиль. Использование встроенного метода предоставляет максимальный контроль над всеми свойствами Web-страницы. Предположим, вы хотите задать внешний вид отдельного абзаца. Вы можете просто добавить атрибут style к тегу абзаца, и браузер отобразит этот абзац с помощью параметров стиля, добавленного в код. У данного метода самый высокий приоритет исполнения, т.е свойства, указанные у атрибута style буду исполнятся в первую очередь, o Внедренный (Embedded). Внедрение позволяет контролировать всю страницу HTML. При использовании тега

style>, помещенного внутри раздела

head> страницы HTML, в код вставляются детализированные атрибуты стиля, которые будут применяться ко всей странице. o Связанный (Linked или External). Связанная таблица стилей - мощный инструмент, который позволяет создавать образцы стилей (master styles), которые можно затем применять ко всему узлу. Основной документ таблицы стилей (расширение .css) создается Web-дизайнером. Этот документ содержит стили, которые будут едиными для всего Web-узла (неважно, содержит одну страницу или тысячи страниц). Любая страница, связанная с этим документом, будет использовать указанные стили.
